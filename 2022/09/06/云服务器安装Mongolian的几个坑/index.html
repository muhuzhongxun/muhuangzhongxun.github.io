云服务器安装Mongolian的几个坑

2022年09月06日 273 字 大概 1 分钟

docker exec -it mongo mongo admin

1.执行指令：docker exec -it mongo mongo admin。

提示：OCI runtime exec failed: exec failed: unable to start container process: exec: “mongo”: executable file not found in $PATH: unknown

原因是官网改用新的命令来创建MongoDB实例Docker Hub

2.navicat for prenium 连接mongo报错：

Cannot connect to MongoDB. No suitable servers found:serverSelectionTimeoutMS expired:[Failed to connect to target host::27017]

进度排查：

  • mongo容器创建

  • 宝塔和阿里云防火墙开放27017端口

  • ip、端口、数据库名、用户密码反复尝试无误

  • navicat连接mongo

还是找不到问题，只能从报错信息下手了。

翻译大法：没有找到合适的服务器:serverSelectionTimeoutMS过期。

突然意识到！！！宝塔虽然开放了端口但是27017的端口并没有被使用，火速查看linux端口占用情况。

netstat -ntlp 发现27017端口没有显示。

docker ps -a 发现mongo与其他镜像不一样的地方在哪了。原来是mongo好像没有做主机与容器端口的映射（官网坑莎沃野！！）
